# Техническое Задание (ТЗ)

**Проект:** Мобильный поздравительный лендинг «Happy Birthday, Babe»

**Тип:** Одностраничное веб-приложение (Single Page Landing)

**Главная цель:** Эмоциональное погружение пользователя через гибридный визуал, аудио-сюрприз и анимацию без лагов на мобильных устройствах.

---

## 1. Общие требования и стек

* **Подход:** **Mobile-First**. Базовый вьюпорт проектирования — `390×844 px` (пропорция `9:16`). Поддержка вертикальной ориентации всех современных смартфонов (iOS / Android).
* **Стек разметки и стилей:** HTML5 / CSS3 (Flexbox/Grid, CSS-анимации) / Vanilla JS (или легкий компонентный фреймворк с деплоем в статику).
* **Деплой:** Vercel / Netlify / GitHub Pages (статическая генерация, HTTPS обязательно для корректной работы автоплея и медиа-апи).
* **Производительность:**
* Вес начальной загрузки страницы (HTML + CSS + JS + фон) — **до 400 КБ**.
* Плавность анимаций — **стабильные 60 FPS**.

---

## 2. Графическая подсистема (Слои фона)

Фон реализуется методом слоеного наложения (Z-index stacking):

1. **Z-index: 1 (Статический базовый фон)**
* Изображение ночного Парижа с Эйфелевой башней, луной и каруселью.
* Формат: `.webp` (сжатие 80%, вес до 200 КБ).
* Стили: `position: fixed; top: 0; left: 0; width: 100vw; height: 100vh; background-size: cover; background-position: center bottom;`.


2. **Z-index: 2 (Слой «Paris Sparkle»)**
* Прозрачный `.png` контур золотых огней строго по координатам Эйфелевой башни.
* Анимация: Бесконечный цикл пульсации прозрачности (`opacity: 0.15` $\rightarrow$ `opacity: 0.85`).
* Timing function: `3s ease-in-out infinite`.


3. **Z-index: 3 (Слой частиц — Лепестки роз)**
* Скрипт генерации падающих элементов кодом (не более **14 штук** в DOM одновременно).
* Форма: Реализуется через CSS `border-radius: 15% 85% 15% 85%` и белый полупрозрачный фон.
* Траектория: Плавное падение сверху вниз (`translateY`) с легким покачиванием по горизонтали (`translateX`) и вращением (`rotateZ`). Отключение интерактивности (`pointer-events: none`).


4. **Z-index: 4 (Слой читаемости / Overlay)**
* Подложка под контентные секции.
* Стили контейнеров контента: `background: rgba(15, 23, 42, 0.65); backdrop-filter: blur(10px); -webkit-backdrop-filter: blur(10px); border: 1px solid rgba(255, 255, 255, 0.12); border-radius: 24px;`.

## 3. Информационная структура (Контентные блоки)

Страница выстраивается в один вертикальный скролл без резких скачков (`scroll-behavior: smooth`).

### Секция 1: Hero (Экран приветствия)

* **Заголовок H1:** `Happy Birthday, my Sayang!`
* **Подзаголовок:** `Today is all about you.`
* **Поведение:** Плавное появление текста из прозрачности при загрузке страницы (Fade-In за `1.5s`).
* **Элемент навигации:** Ненавязчивый анимированный индикатор скролла вниз (пульсирующая стрелочка или текст *«Scroll down gently»*).

### Секция 2: The Letter (Личное послание)

* **Текст:** Утвержденный английский текст о поддержке, восхищении её улыбкой и преодолении расстояния.
* **Типографика:**
* Шрифт: Чистый современный Sans-Serif (например, *Inter*, *Outfit* или *Montserrat*).
* Размер: `16px–18px`.
* Межстрочный интервал (line-height): `1.6` (важно для комфортного чтения не родного языка).
* Цвет: `rgba(255, 255, 255, 0.92)`.



### Секция 3: Audio Surprise (Песня от сердца)

* **Подводка:** `I wanted to give you something you can listen to whenever you miss me...`
* **Кастомный аудиоплеер:**
* Запрет использования стандартного (некрасивого) HTML5-плеёра браузера.
* Элементы интерфейса: Крупная круглая кнопка **Play / Pause** (белый матовый круг с иконкой), тонкая линия прогресс-бара, таймер текущего времени трека.
* Метаданные трека на экране: `Eugen & Irene — Sayang`.
* *Прим по логике:* Аудиофайл `.mp3` подгружается асинхронно по нажатию кнопки Play, чтобы не тормозить первоначальную отрисовку сайта.



### Секция 4: Future Vision (Видео Чиби)

* **Подводка:** `And one last surprise... Just you, me, and your favorite blue sneakers.`
* **Видеоплеер:**
* Контейнер с соотношением сторон `9:16`, вписанный в карточку со скругленными углами.
* Атрибуты тега `<video>`: `playsinline`, `preload="metadata"`.
* Постер (кадр заглушки): Выбранный самый милый стоп-кадр из Чиби-анимации.
* Запуск: Кнопка Play по центру постера.

## 4. UI/UX и поведенческие сценарии

* **Правило одной руки:** Все интерактивные кнопки (Play аудио, Play видео) должны располагаться в так называемой *Thumb Zone* (средняя и нижняя треть экрана), чтобы ей не приходилось тянуться пальцем к верхнему краю телефона.
* **Предотвращение ошибок скролла:** Жесткая блокировка горизонтального смещения контейнера страницы:
```css
html, body {
  max-width: 100%;
  overflow-x: hidden;
}

```


* **Мягкость границ:** Отступы внутри карточек контента (`padding`) — не менее `24px`. Внешние отступы между блоками (`margin-bottom`) — `48px–64px`, чтобы каждый блок воспринимался как отдельный этап квеста.

## 5. Пошаговый план сборки (Pipeline)

1. **Подготовка медиа:**
* Получение фонового арта из нейросети $\rightarrow$ компрессия в `bg-paris.webp`.
* Вырезание маски огней в Фотошопе $\rightarrow$ сохранение в `sparkle.png`.


2. **Верстка каркаса:**
* Сборка HTML-структуры + наложение CSS-блюр карточек поверх статического фона.


3. **Программирование динамики:**
* Подключение JS-скрипта падающих лепестков роз.
* Верстка кастомного JS-контроллера для тега `<audio>`.


4. **Интеграция контента:**
* Вставка финального английского послания и готовой ИИ-песни.


5. **Тестирование и деплой:**
* Проверка в режиме эмуляции мобильных устройств (на iOS Safari особое внимание на корректность работы `backdrop-filter: blur()`). Публикация на хостинг.
